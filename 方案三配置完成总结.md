## 📊 最终架构

### 线程架构（3个线程）
```
1. 主线程（ros2_control 控制循环）
   - 调用 read() 函数
   - 发布 /imu/rpy 话题
   
2. 电机串口线程
   - 读取电机数据
   
3. IMU串口线程
   - 读取IMU数据
   - 调用 parseImuData() 更新内存变量
```

### 数据流
```
IMU硬件 → IMU串口线程 → parseImuData()
                              ↓
                        更新内存变量
                              ↓
                    ┌─────────┴─────────┐
                    ↓                   ↓
            export_state_interfaces()  read()函数
                    ↓                   ↓
            IMUSensorBroadcaster    发布/imu/rpy
                    ↓
            /imu_sensor_broadcaster/imu
```

### 发布的话题
```
/imu_sensor_broadcaster/imu    # sensor_msgs/Imu（四元数、角速度、加速度）
/imu/rpy                        # geometry_msgs/Vector3Stamped（欧拉角，角度）
```

### 导出的状态接口（13个）
```
imu_sensor/orientation.x           # 四元数 x
imu_sensor/orientation.y           # 四元数 y
imu_sensor/orientation.z           # 四元数 z
imu_sensor/orientation.w           # 四元数 w
imu_sensor/rpy.roll                # 欧拉角 roll（弧度）
imu_sensor/rpy.pitch               # 欧拉角 pitch（弧度）
imu_sensor/rpy.yaw                 # 欧拉角 yaw（弧度）
imu_sensor/angular_velocity.x      # 角速度 x
imu_sensor/angular_velocity.y      # 角速度 y
imu_sensor/angular_velocity.z      # 角速度 z
imu_sensor/linear_acceleration.x   # 加速度 x
imu_sensor/linear_acceleration.y   # 加速度 y
imu_sensor/linear_acceleration.z   # 加速度 z
```

```cpp
class MyController : public controller_interface::ControllerInterface {
public:
    controller_interface::InterfaceConfiguration 
    state_interface_configuration() const override {
        controller_interface::InterfaceConfiguration config;
        config.type = controller_interface::interface_configuration_type::INDIVIDUAL;
        
        config.names = {
            "imu_sensor/rpy.roll",              // 欧拉角（弧度）
            "imu_sensor/rpy.pitch",
            "imu_sensor/rpy.yaw",
            "imu_sensor/angular_velocity.x",    // 角速度
            "imu_sensor/angular_velocity.y",
            "imu_sensor/angular_velocity.z",
        };
        
        return config;
    }
    
    controller_interface::return_type update(
        const rclcpp::Time& time,
        const rclcpp::Duration& period) override 
    {
        // 直接读取，零延迟！
        double roll = state_interfaces_[0].get_value();   // 弧度
        double pitch = state_interfaces_[1].get_value();
        double yaw = state_interfaces_[2].get_value();
        
        // 使用这些数据进行控制
        compute_control(roll, pitch, yaw);
        
        return controller_interface::return_type::OK;
    }
};
```

## 🔧 常见问题

### Q1: 找不到 imu_sensor_broadcaster

**A:** 安装依赖：
```bash
sudo apt install ros-humble-imu-sensor-broadcaster
```

### Q2: 话题 /imu_sensor_broadcaster/imu 没有数据

**A:** 检查：
1. IMU 串口是否正确连接（`/dev/ttyUSB0` 或配置的端口）
2. URDF 中是否声明了所有13个状态接口
3. 查看日志：`ros2 topic echo /rosout`

### Q3: /imu/rpy 话题没有数据

**A:** 检查：
1. URDF 中 `publish_rpy` 参数是否为 `true`
2. IMU 是否正在发送 RPY 数据包（rid == 0x03）
3. 查看硬件接口日志

### Q4: 控制器读取不到 IMU 状态接口

**A:** 确保：
1. URDF 中声明了对应的状态接口
2. 控制器的 `state_interface_configuration()` 中声明了需要的接口
3. 接口名称完全匹配（例如 `imu_sensor/rpy.roll`）

### Q5: 编译错误

**A:** 确保：
1. 已安装 `imu_sensor_broadcaster` 包
2. 已 source ROS2 环境：`source /opt/ros/humble/setup.bash`
3. 清理后重新编译：`rm -rf build install log && colcon build`
