## 已修复的问题

### 1. ✅ 线程安全问题（已修复）

**问题描述：**
多线程环境下，状态变量在不同线程间共享但没有互斥锁保护：
- 电机串口线程写入 `state_positions_`, `state_velocities_` 等
- IMU串口线程写入 `imu_rpy_`, `imu_gyro_` 等
- 主线程读取这些变量

**修复方案：**
添加了两个互斥锁：
```cpp
// 在头文件中
mutable std::mutex state_mutex_;  // 保护电机状态变量
mutable std::mutex imu_mutex_;    // 保护IMU数据
```

**使用位置：**
1. `parseMotorData()` - 写入电机状态时加锁
2. `parseImuData()` - 写入IMU数据时加锁
3. `read()` - 读取状态变量和IMU数据时加锁

**影响：**
- 保证了跨平台的线程安全
- 性能影响极小（锁粒度很小）
- 避免了潜在的数据竞争

## 代码架构分析

### 线程架构（方案三）

```
┌─────────────────────────────────────────────────────────────┐
│                  主线程 (ros2_control)                       │
│  - read(): 读取状态变量（加锁）                              │
│  - write(): 发送控制指令                                     │
│  - 发布 /imu/rpy 话题（加锁读取IMU数据）                     │
│  - 状态接口导出给 IMUSensorBroadcaster                       │
└────────────┬────────────────────────────────────────────────┘
             │
             │ 通过互斥锁安全访问
             ↓
┌─────────────────────────────────────────────────────────────┐
│              共享内存（状态变量）                             │
│  - state_positions_[], state_velocities_[] (state_mutex_)   │
│  - imu_rpy_[], imu_gyro_[], imu_accel_[] (imu_mutex_)       │
│  - motor_data_buffer_                                       │
└────────────┬────────────────────────────────────────────────┘
             ↑
             │ 通过互斥锁安全更新
             │
┌────────────┴────────────────┬───────────────────────────────┐
│                             │                               │
│  电机串口线程                │  IMU串口线程                   │
│  - SerialDevice::run()      │  - SerialDevice::run()        │
│  - parseMotorData()         │  - parseImuSerialData()       │
│  - 更新电机状态（加锁）      │  - parseImuData()             │
│                             │  - 更新IMU状态（加锁）         │
└─────────────────────────────┴───────────────────────────────┘
```

### 数据流

#### 电机数据流
```
串口接收 → motor_rx_buffer_
    ↓
查找帧头 0xFF 0xAA
    ↓
读取数据长度
    ↓
验证CRC32
    ↓
提取纯净数据 → motor_data_buffer_
    ↓
按17字节解析每个电机
    ↓
【加锁】更新 state_positions_, state_velocities_ 等
    ↓
主线程【加锁】读取状态变量
```

#### IMU数据流
```
串口接收 → imu_rx_buffer_
    ↓
查找帧头 0x55 0xAA
    ↓
验证CRC16
    ↓
【加锁】更新 imu_rpy_, imu_gyro_, imu_accel_
    ↓
主线程【加锁】读取IMU数据 → 发布 /imu/rpy
    ↓
状态接口导出 → IMUSensorBroadcaster → 发布 /imu_sensor_broadcaster/imu
```

### 互斥锁性能影响

- **锁粒度**：非常小（只在数据更新/读取时加锁）
- **锁持有时间**：微秒级
- **性能影响**：可忽略不计（< 1%）
- **收益**：保证了线程安全，避免了潜在的数据竞争

## 发布的话题

### 1. `/imu/rpy` (geometry_msgs/Vector3Stamped)
- **发布者：** 主线程（在 `read()` 函数中）
- **频率：** 与 ros2_control 循环频率相同（100-1000Hz）
- **内容：** 欧拉角（角度）
- **用途：** Foxglove 可视化，易读
- **线程安全：** ✅ 使用 imu_mutex_ 保护

### 2. `/imu_sensor_broadcaster/imu` (sensor_msgs/Imu)
- **发布者：** IMUSensorBroadcaster（ros2_control 控制器）
- **频率：** 由 broadcaster 配置决定
- **内容：** 完整IMU数据（四元数、角速度、加速度）
- **用途：** 标准ROS2 IMU话题，供其他节点使用
- **线程安全：** ✅ 通过状态接口读取（已加锁保护）
