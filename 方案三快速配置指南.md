## 配置步骤

### 步骤1：修改 URDF 文件

找到你的机器人 URDF 文件（例如 `infantry_real.xacro`），在 `<ros2_control>` 标签内添加 IMU 传感器定义：

```xml
<ros2_control name="TideSystem" type="system">
    <hardware>
        <plugin>tide_hw_interface/TideHardwareInterface</plugin>
        <param name="serial_port">/dev/ttyUSB0</param>
        <param name="imu_serial_port">/dev/ttyUSB1</param>
        <param name="baud_rate">115200</param>
        <param name="imu_baud_rate">921600</param>
        <param name="enable_virtual_control">false</param>
        <param name="need_calibration">false</param>
        <param name="publish_rpy">true</param>
        <param name="imu_frame_id">imu_link</param>
    </hardware>
    
    <!-- 现有的关节定义 -->
    <joint name="joint1">
        <!-- ... -->
    </joint>
    
    <!-- 新增：IMU传感器定义 -->
    <sensor name="imu_sensor">
        <!-- 四元数 (4个) -->
        <state_interface name="orientation.x"/>
        <state_interface name="orientation.y"/>
        <state_interface name="orientation.z"/>
        <state_interface name="orientation.w"/>
        
        <!-- 欧拉角 (3个) -->
        <state_interface name="rpy.roll"/>
        <state_interface name="rpy.pitch"/>
        <state_interface name="rpy.yaw"/>
        
        <!-- 角速度 (3个) -->
        <state_interface name="angular_velocity.x"/>
        <state_interface name="angular_velocity.y"/>
        <state_interface name="angular_velocity.z"/>
        
        <!-- 线性加速度 (3个) -->
        <state_interface name="linear_acceleration.x"/>
        <state_interface name="linear_acceleration.y"/>
        <state_interface name="linear_acceleration.z"/>
    </sensor>
</ros2_control>
```

## 控制器中使用 IMU 数据

如果你想在自定义控制器中直接读取 IMU 数据（零延迟）：

```cpp
class MyController : public controller_interface::ControllerInterface {
public:
    controller_interface::InterfaceConfiguration 
    state_interface_configuration() const override {
        controller_interface::InterfaceConfiguration config;
        config.type = controller_interface::interface_configuration_type::INDIVIDUAL;
        
        // 声明需要的IMU状态接口
        config.names = {
            "imu_sensor/rpy.roll",              // 欧拉角（弧度）
            "imu_sensor/rpy.pitch",
            "imu_sensor/rpy.yaw",
            "imu_sensor/angular_velocity.x",    // 角速度
            "imu_sensor/angular_velocity.y",
            "imu_sensor/angular_velocity.z",
        };
        
        return config;
    }
    
    controller_interface::return_type update(
        const rclcpp::Time& time,
        const rclcpp::Duration& period) override 
    {
        // 直接读取，零延迟！
        double roll = state_interfaces_[0].get_value();   // 弧度
        double pitch = state_interfaces_[1].get_value();
        double yaw = state_interfaces_[2].get_value();
        
        double gyro_x = state_interfaces_[3].get_value();
        double gyro_y = state_interfaces_[4].get_value();
        double gyro_z = state_interfaces_[5].get_value();
        
        // 使用这些数据进行控制
        compute_control(roll, pitch, yaw, gyro_x, gyro_y, gyro_z);
        
        return controller_interface::return_type::OK;
    }
};
```

## 常见问题

### Q1: 找不到 imu_sensor_broadcaster

**A:** 安装依赖：
```bash
sudo apt install ros-humble-imu-sensor-broadcaster
```

### Q2: 话题 /imu_sensor_broadcaster/imu 没有数据

**A:** 检查：
1. IMU 串口是否正确连接
2. URDF 中是否声明了所有13个状态接口
3. 查看日志：`ros2 topic echo /rosout`

### Q3: /imu/rpy 话题没有数据

**A:** 检查：
1. URDF 中 `publish_rpy` 参数是否为 `true`
2. IMU 是否正在发送 RPY 数据包（rid == 0x03）

### Q4: 控制器读取不到 IMU 状态接口

**A:** 确保：
1. URDF 中声明了对应的状态接口
2. 控制器的 `state_interface_configuration()` 中声明了需要的接口
3. 接口名称完全匹配（例如 `imu_sensor/rpy.roll`）
