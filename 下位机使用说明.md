# 下位机通信协议使用说明

## 协议概述

新协议使用 `0xFF 0xAA` 帧头和 CRC32 校验，支持灵活的数据长度。

### 数据包格式
```
[0xFF][0xAA][长度(2字节)][数据(N字节)][CRC32(4字节)]
```

## 快速开始

### 1. 初始化

```c
#include "trans_task.h"

void main(void)
{
    // 初始化通信
    trans_task_init();
    
    // 在主循环中调用
    while(1)
    {
        trans_control_task();
        // ...
    }
}
```

### 2. 发送数据（3种方式）

#### 方式1：发送单个电机数据（最常用）

```c
// 发送电机1的数据
send_single_motor(
    1,          // 电机ID
    25.0f,      // 位置
    0.5f,       // 速度
    2.3f,       // 电流
    45.5f       // 温度
);

// 数据包：17字节
// [ID(1)] + [位置(4)] + [速度(4)] + [电流(4)] + [温度(4)]
```

#### 方式2：批量发送多个电机（推荐）

```c
// 准备3个电机的数据
MotorData_t motors[3] = {
    {1, 25.0f, 0.5f, 2.3f, 45.5f},  // 电机1
    {2, 30.0f, 0.3f, 1.8f, 42.0f},  // 电机2
    {3, 15.0f, 0.8f, 3.1f, 48.2f}   // 电机3
};

// 一次性发送
send_multiple_motors(motors, 3);

// 数据包：64字节
// [数量(1)] + [电机1数据(17)] + [电机2数据(17)] + [电机3数据(17)]
```

#### 方式3：发送浮点数数组（最简单）

```c
// 直接发送浮点数数组
float data[] = {25.0f, 0.5f, 2.3f, 45.5f, 30.0f, 0.3f};
send_float_array(data, 6);

// 数据包：24字节（6个float）
```

### 3. 发送自定义数据

```c
// 自己组织数据
uint8_t custom_data[50];
size_t offset = 0;

// 添加一个uint8_t
custom_data[offset++] = 0x01;

// 添加一个float
float value = 123.45f;
memcpy(custom_data + offset, &value, sizeof(float));
offset += sizeof(float);

// 添加更多数据...

// 发送
send_custom_data(custom_data, offset);
```

### 示例4：替换原有的Send_to_pc函数

```c
// 原来的函数
void Send_to_pc_old(RpyTypeDef data_r)
{
    pack_Rpy(&data_r, ins.pitch, 88.8f, openfire, team_color);
    Check_Rpy(&data_r);
    CDC_Transmit_HS((uint8_t*)&data_r, sizeof(data_r));
}

// 新的函数（更简单）
void Send_to_pc_new(void)
{
    // 方式1：发送浮点数数组
    float data[] = {
        ins.pitch,
        88.8f,
        openfire,
        (float)team_color
    };
    send_float_array(data, 4);
    
    // 或者方式2：自定义格式
    uint8_t custom_data[20];
    size_t offset = 0;
    
    memcpy(custom_data + offset, &ins.pitch, sizeof(float)); offset += sizeof(float);
    float temp = 88.8f;
    memcpy(custom_data + offset, &temp, sizeof(float)); offset += sizeof(float);
    memcpy(custom_data + offset, &openfire, sizeof(float)); offset += sizeof(float);
    custom_data[offset++] = (uint8_t)team_color;
    
    send_custom_data(custom_data, offset);
}
```

## 接收数据

### 在USB接收回调中调用

```c
// 在 usbd_cdc_if.c 中
static int8_t CDC_Receive_HS(uint8_t* Buf, uint32_t *Len)
{
    // 调用处理函数
    process_usb_data(Buf, Len);
    
    USBD_CDC_SetRxBuffer(&hUsbDeviceHS, &Buf[0]);
    USBD_CDC_ReceivePacket(&hUsbDeviceHS);
    return (USBD_OK);
}
```

## 数据长度对照表

| 发送方式 | 数据长度 | 总包长度 | 说明 |
|---------|---------|---------|------|
| 单个电机 | 17字节 | 27字节 | 1+4+4+4+4 |
| 3个电机批量 | 52字节 | 62字节 | 1+17×3 |
| 4个浮点数 | 16字节 | 26字节 | 4×4 |
| 10个浮点数 | 40字节 | 50字节 | 10×4 |

总包长度 = 帧头(2) + 长度(2) + 数据(N) + CRC32(4) = N + 8

## 迁移指南

### 从旧协议迁移

```c
// 旧代码
RpyTypeDef rpy_tx_data;
pack_Rpy(&rpy_tx_data, yaw, pitch, roll, team_color);
Check_Rpy(&rpy_tx_data);
CDC_Transmit_HS((uint8_t*)&rpy_tx_data, sizeof(rpy_tx_data));

// 新代码（更简单）
float data[] = {yaw, pitch, roll, (float)team_color};
send_float_array(data, 4);
```

### 优势对比

| 项目 | 旧协议 | 新协议 |
|------|--------|--------|
| 帧头 | 0xFF | 0xFF 0xAA |
| 校验 | 和校验+附加校验 | CRC32 |
| 数据长度 | 固定 | 灵活 |
| 使用便捷性 | 需要pack和check | 一行代码搞定 |
| 可靠性 | 较低 | 高 |

## 常见问题

### Q1: 如何发送超过256字节的数据？
A: 修改 `MOTOR_MAX_DATA_LEN` 宏定义，或者分多次发送。

### Q2: 如何知道发送成功？
A: USB CDC会返回状态，可以检查 `CDC_Transmit_HS` 的返回值。

### Q3: 接收数据如何解析？
A: 在 `process_usb_data` 函数中修改解析逻辑，根据你的数据格式解析。

### Q4: 可以混合发送不同格式的数据吗？
A: 可以！建议在数据开头添加一个类型字节来区分。

```c
// 示例：带类型标识的数据
uint8_t data[100];
data[0] = 0x01;  // 类型1：单个电机
// ... 填充电机数据

data[0] = 0x02;  // 类型2：IMU数据
// ... 填充IMU数据
```
