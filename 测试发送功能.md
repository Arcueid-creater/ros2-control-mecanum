## 数据流程（现在完整了）

```
你的代码调用
    ↓
sendCustomData() / sendFloatArray()
    ↓
buildCustomPacket() (自动打包：帧头 + 长度 + 数据 + CRC32)
    ↓
serial_device_->write(packet)
    ↓
SerialDevice::write() (加入发送队列)
    ↓
write_queue_.push(data)
    ↓
write_cv_.notify_one() (通知发送线程)
    ↓
SerialDevice::run() 线程被唤醒
    ↓
const auto& data = write_queue_.front()  ✅ 现在会执行
    ↓
::write(fd_, data.data(), data.size())   ✅ 实际发送到串口
    ↓
write_queue_.pop()                        ✅ 移除已发送数据
    ↓
下位机 USB CDC 接收
    ↓
process_usb_data() 解析
```

```cpp
  if (data.empty()) return;
```

## 预期结果

### 发送4个浮点数 `{1.23f, 4.56f, 7.89f, 10.11f}`

**数据包内容（十六进制）：**
```
FF AA          # 帧头
10 00          # 数据长度 = 16字节 (4个float)
A4 70 9D 3F    # 1.23 (float, 小端序)
B8 1E 91 40    # 4.56
F6 28 FC 40    # 7.89
8F C2 21 41    # 10.11
XX XX XX XX    # CRC32 (4字节)
```

**总包大小：** 8 + 16 = 24字节

### 发送字节数组 `{0x01, 0x02, 0x03, 0x04, 0x05}`

**数据包内容：**
```
FF AA          # 帧头
05 00          # 数据长度 = 5字节
01 02 03 04 05 # 数据
XX XX XX XX    # CRC32
```

**总包大小：** 8 + 5 = 13字节

## 常见问题排查

### 问题1：数据没有发送

**检查项：**
1. ✅ 确认 `SerialDevice::run()` 中的发送代码已取消注释
2. ✅ 确认串口已正确打开（检查 `serial_device_` 不为空）
3. ✅ 确认 `running_` 标志为 true
4. ✅ 确认线程正在运行

**调试代码：**
```cpp
void SerialDevice::write(const std::vector<uint8_t>& data)
{
  std::lock_guard<std::mutex> lock(write_mutex_);
  write_queue_.push(data);
  printf("📤 Added %zu bytes to write queue (queue size: %zu)\n", 
         data.size(), write_queue_.size());
  write_cv_.notify_one();
}
```

### 问题2：下位机收不到数据

**检查项：**
1. 串口连接是否正常
2. 波特率是否匹配（默认921600）
3. 下位机的 `process_usb_data()` 是否被调用
4. CRC32 计算是否正确

**测试代码（下位机）：**
```c
void CDC_Receive_HS(uint8_t* Buf, uint32_t *Len)
{
    printf("📥 Received %lu bytes from USB\r\n", *Len);
    process_usb_data(Buf, Len);
    
    USBD_CDC_SetRxBuffer(&hUsbDeviceHS, &Buf[0]);
    USBD_CDC_ReceivePacket(&hUsbDeviceHS);
    return (USBD_OK);
}
```

### 问题3：CRC32 校验失败

**原因：** 上下位机的 CRC32 算法不一致

**验证：** 使用相同的测试数据计算 CRC32

```cpp

printf("CRC32: 0x%08X\n", crc);
```


## 性能监控

### 监控发送队列大小

```cpp
void SerialDevice::run()
{
  while (running_ && rclcpp::ok())
  {
    {
      std::unique_lock<std::mutex> lock(write_mutex_);
      if (write_cv_.wait_for(lock, std::chrono::milliseconds(1), 
          [this] { return !write_queue_.empty() || !running_; }))
      {
        if (!running_) break;
        
        size_t queue_size = write_queue_.size();
        if (queue_size > 10)
        {
          printf("⚠️  Write queue is large: %zu packets\n", queue_size);
        }
        
        while (!write_queue_.empty())
        {
          const auto& data = write_queue_.front();
          ssize_t written = ::write(fd_, data.data(), data.size());
          
          if (written < 0)
          {
            printf("❌ Write error: %s\n", strerror(errno));
          }
          else if (written != static_cast<ssize_t>(data.size()))
          {
            printf("⚠️  Partial write: %zd/%zu bytes\n", written, data.size());
          }
          
          write_queue_.pop();
        }
      }
    }
    // ... 接收部分
  }
}
```
