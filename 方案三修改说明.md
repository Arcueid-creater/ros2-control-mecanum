### 3. 修改 `parseImuData()` 函数

**位置：** `src/tide_control/tide_hw_interface/src/tide_hw_interface.cpp` 第565-600行

**修改内容：**
- ❌ 删除所有发布逻辑（`imu_pub_->publish()` 和 `rpy_pub_->publish()`）
- ✅ 只更新内存变量（`imu_rpy_`, `imu_gyro_`, `imu_accel_`, `imu_orientation`）
- ✅ 发布逻辑移到 `read()` 函数中

**修改后代码：**
```cpp
void TideHardwareInterface::parseImuData(const uint8_t* frame)
{
  ImuPacket packet;
  std::memcpy(&packet, frame, sizeof(ImuPacket));

  // 方案三：只更新内存变量，不在回调中发布
  // 发布逻辑移到 read() 函数中
  
  if (packet.rid == 0x03) // RPY
  {
    imu_rpy_[0] = packet.data[0] * M_PI / 180.0;  // 转为弧度
    imu_rpy_[1] = packet.data[1] * M_PI / 180.0;
    imu_rpy_[2] = packet.data[2] * M_PI / 180.0;
  }
  else if (packet.rid == 0x02) // Gyro
  {
    imu_gyro_[0] = packet.data[0] * M_PI / 180.0;
    imu_gyro_[1] = packet.data[1] * M_PI / 180.0;
    imu_gyro_[2] = packet.data[2] * M_PI / 180.0;
  }
  else if (packet.rid == 0x01) // Accel
  {
    imu_accel_[0] = packet.data[0]; // Assuming m/s^2
    imu_accel_[1] = packet.data[1];
    imu_accel_[2] = packet.data[2];
  }
  else if (packet.rid == 0x04) // Quaternion
  {
    imu_orientation[0] = packet.data[0];  // w
    imu_orientation[1] = packet.data[1];  // x
    imu_orientation[2] = packet.data[2];  // y
    imu_orientation[3] = packet.data[3];  // z
  }
}
```

### 4. 修改 `read()` 函数

**位置：** `src/tide_control/tide_hw_interface/src/tide_hw_interface.cpp` 第858-900行

**修改内容：**
- ✅ 在主线程中发布 `/imu/rpy` 话题
- ✅ 使用 `nh_->now()` 获取时间戳
- ✅ 将欧拉角转为角度（方便 Foxglove 显示）

```cpp
// 方案三：在主线程中发布RPY话题
// IMU数据已经在 parseImuData() 中更新到内存变量
// 状态接口会自动读取这些变量
if (publish_rpy_ && rpy_pub_)
{
  auto rpy_msg = geometry_msgs::msg::Vector3Stamped();
  rpy_msg.header.stamp = nh_->now();
  rpy_msg.header.frame_id = imu_frame_id_;
  // 转为角度（方便Foxglove显示）
  rpy_msg.vector.x = imu_rpy_[0] * 180.0 / M_PI;  // roll (degrees)
  rpy_msg.vector.y = imu_rpy_[1] * 180.0 / M_PI;  // pitch (degrees)
  rpy_msg.vector.z = imu_rpy_[2] * 180.0 / M_PI;  // yaw (degrees)
  rpy_pub_->publish(rpy_msg);
}
```

## 线程架构变化

### 修改前（4个线程）
```
1. 主线程（ros2_control 控制循环）
2. 电机串口线程
3. IMU串口线程
4. IMU spin线程 ← 删除
```

### 修改后（3个线程）
```
1. 主线程（ros2_control 控制循环）
2. 电机串口线程
3. IMU串口线程
```

## 数据流变化

### 修改前
```
IMU硬件 → IMU串口线程 → parseImuData()
                              ↓
                        直接发布话题（需要spin线程）
                              ↓
                        /imu/data, /imu/rpy
```

### 修改后
```
IMU硬件 → IMU串口线程 → parseImuData()
                              ↓
                        更新内存变量
                              ↓
                    ┌─────────┴─────────┐
                    ↓                   ↓
            export_state_interfaces()  read()函数
                    ↓                   ↓
            IMUSensorBroadcaster    发布/imu/rpy
                    ↓
            /imu_sensor_broadcaster/imu
```
