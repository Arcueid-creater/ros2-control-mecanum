# 缓冲区大小配置说明

## 修改内容

已将 `MOTOR_MAX_DATA_LEN` 从 256 字节增加到 1024 字节，并相应调整了相关缓冲区。

---

## 配置参数

### 当前配置（已修改）

```c
// trans_task_new.h
#define MOTOR_MAX_DATA_LEN      1024    // 最大数据长度

// trans_task_new.c
static uint8_t rx_buffer[2048];              // 接收缓冲区
static uint8_t received_data_buffer[1024];   // 数据存储（自动使用 MOTOR_MAX_DATA_LEN）
static uint8_t tx_buffer[2048];              // 发送缓冲区（在 send_packet 函数中）
```

## 容量分析

### 数据包结构

```
完整数据包 = 帧头(2) + 长度(2) + 数据(N) + CRC32(4)
```

### 不同配置下的容量

| MOTOR_MAX_DATA_LEN | 最大数据包大小 | 可传输电机数 | 可传输浮点数 | 内存占用 |
|-------------------|--------------|------------|------------|---------|
| 256 字节（旧）     | 270 字节     | 15 个      | 64 个      | ~1 KB   |
| 512 字节          | 526 字节     | 30 个      | 128 个     | ~2 KB   |
| **1024 字节（当前）** | **1038 字节** | **60 个** | **256 个** | **~4 KB** |
| 2048 字节         | 2062 字节    | 120 个     | 512 个     | ~8 KB   |

**计算说明：**
- 每个电机数据 = ID(1) + position(4) + velocity(4) + current(4) + temperature(4) = 17 字节
- 每个浮点数 = 4 字节

## 推荐配置

### 1. 小型系统（≤15个电机）
```c
#define MOTOR_MAX_DATA_LEN      256
static uint8_t rx_buffer[512];
static uint8_t tx_buffer[512];
```
- 内存占用：~1 KB
- 适用场景：简单机器人、测试系统

### 2. 中型系统（≤30个电机）✅ 推荐
```c
#define MOTOR_MAX_DATA_LEN      512
static uint8_t rx_buffer[1024];
static uint8_t tx_buffer[1024];
```
- 内存占用：~2 KB
- 适用场景：大多数机器人系统

### 3. 大型系统（≤60个电机）⭐ 当前配置
```c
#define MOTOR_MAX_DATA_LEN      1024
static uint8_t rx_buffer[2048];
static uint8_t tx_buffer[2048];
```
- 内存占用：~4 KB
- 适用场景：复杂机器人、多传感器系统

### 4. 超大型系统（≤120个电机）
```c
#define MOTOR_MAX_DATA_LEN      2048
static uint8_t rx_buffer[4096];
static uint8_t tx_buffer[4096];
```
- 内存占用：~8 KB
- 适用场景：工业机器人、多机协同

## 硬件限制

### STM32 内存限制

| 芯片型号 | RAM大小 | 推荐最大配置 |
|---------|--------|------------|
| STM32F103 | 20-64 KB | 512 字节 |
| STM32F407 | 192 KB | 1024 字节 ✅ |
| STM32F427 | 256 KB | 2048 字节 |
| STM32H743 | 1 MB | 4096 字节 |

### USB CDC 限制

- USB Full Speed (FS): 最大包 64 字节
- USB High Speed (HS): 最大包 512 字节
- 建议单次传输 ≤ 2048 字节（避免分包过多）

### 示例1：发送60个电机数据（需要1024字节）

```c
MotorData_t motors[60];

// 填充电机数据
for (int i = 0; i < 60; i++)
{
    motors[i].motor_id = i + 1;
    motors[i].position = get_motor_position(i);
    motors[i].velocity = get_motor_velocity(i);
    motors[i].current = get_motor_current(i);
    motors[i].temperature = get_motor_temperature(i);

// 发送（需要 1 + 60*17 = 1021 字节）
send_multiple_motors(motors, 60);
```

```c

// 填充传感器数据

{
    sensor_data[i] = read_sensor(i);

// 发送（需要 256*4 = 1024 字节）
send_float_array(sensor_data, 256);
```

### 示例3：发送混合数据

```c
uint8_t custom_data[1024];

    custom_data[offset++] = i + 1;  // motor_id
    
    float pos = get_motor_position(i);
    memcpy(custom_data + offset, &pos, sizeof(float));
    offset += sizeof(float);
    
    // ... 其他数据
}

// 添加传感器数据
for (int i = 0; i < 100; i++)
{
    float sensor = read_sensor(i);
    memcpy(custom_data + offset, &sensor, sizeof(float));
    offset += sizeof(float);
}

// 发送
send_custom_data(custom_data, offset);
```

### 内存占用对比

| 配置 | 静态内存 | 栈内存（最大） | 总计 |
|------|---------|--------------|------|
| 256字节 | ~1 KB | ~0.5 KB | ~1.5 KB |
| 512字节 | ~2 KB | ~1 KB | ~3 KB |

| 数据大小 | 传输时间 | 备注 |
|---------|---------|------|
| 256 字节 | ~0.04 ms | 极快 |
| 512 字节 | ~0.08 ms | 很快 |
| 1024 字节 | ~0.17 ms | 快 |
| 2048 字节 | ~0.34 ms | 正常 |

**结论：** 1024字节配置在性能和容量之间取得了很好的平衡。

## 如何修改配置

### 方法1：修改宏定义（推荐）

只需修改 `trans_task_new.h` 中的一个宏：

```c
// 小型系统
#define MOTOR_MAX_DATA_LEN      256

// 中型系统
#define MOTOR_MAX_DATA_LEN      512

// 大型系统（当前）
#define MOTOR_MAX_DATA_LEN      1024

// 超大型系统
#define MOTOR_MAX_DATA_LEN      2048
```

然后修改 `trans_task_new.c` 中的缓冲区大小：

```c
// 缓冲区大小 = MOTOR_MAX_DATA_LEN * 2（留有余量）
static uint8_t rx_buffer[MOTOR_MAX_DATA_LEN * 2];

// 在 send_packet 函数中
static uint8_t tx_buffer[MOTOR_MAX_DATA_LEN * 2];
```

### 方法2：动态分配（高级）

如果内存紧张，可以使用动态分配：

```c
// 在需要时分配
uint8_t *tx_buffer = malloc(MOTOR_MAX_DATA_LEN + 8);
if (tx_buffer == NULL) {
    // 处理内存不足
    return;
}

// 使用完后释放
free(tx_buffer);
```

## 注意事项

### 1. 内存对齐

确保结构体使用 `__attribute__((packed))`：

```c
typedef struct {
    uint8_t motor_id;
    float position;
    float velocity;
    float current;
    float temperature;
} __attribute__((packed)) MotorData_t;
```

### 2. 栈溢出风险

如果在函数中使用大数组，注意栈大小：

```c
// ❌ 危险：可能导致栈溢出
void some_function(void)
{
    uint8_t large_array[2048];  // 在栈上分配
    // ...
}

// ✅ 安全：使用静态或全局变量
static uint8_t large_array[2048];  // 在静态区分配
```

### 3. USB传输限制

USB CDC 有最大传输单元（MTU）限制：
- USB FS: 64 字节/包
- USB HS: 512 字节/包

大数据会自动分包，但建议：
- 单次传输 ≤ 2048 字节
- 避免频繁发送超大数据包

### 4. 实时性考虑

数据包越大，传输时间越长：
- 256 字节：适合高频率（1000 Hz）
- 512 字节：适合中频率（500 Hz）
- 1024 字节：适合低频率（100-200 Hz）
- 2048 字节：适合低频率（50-100 Hz）

## 总结

### 当前配置（1024字节）

✅ **优点：**
- 可传输60个电机数据或256个浮点数
- 内存占用适中（~6 KB）
- 适合大多数复杂机器人系统
- 传输速度快（~0.17 ms）

⚠️ **注意：**
- 确保STM32有足够RAM（建议≥192 KB）
- 注意栈大小配置（建议≥4 KB）
- 避免在栈上分配大数组

### 如何选择

- **简单系统**：256 字节
- **一般系统**：512 字节
- **复杂系统**：1024 字节 ⭐ 推荐
- **超大系统**：2048 字节
