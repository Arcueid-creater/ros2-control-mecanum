# 控制器如何发送数据给下位机

## 数据流程

```
ROS2 Controller (控制器)
    ↓ 设置命令
cmd_positions_[i] = 1.23
cmd_velocities_[i] = 0.5
    ↓ 每个控制周期调用
TideHardwareInterface::write()
    ↓ 在这里发送自定义数据
sendFloatArray() / sendCustomData()
    ↓ 自动打包
[0xFF 0xAA][长度][数据][CRC32]
    ↓ 串口发送
下位机接收
```

## 关键点

1. **控制器不能直接调用发送函数**
   - 控制器只能通过 `cmd_positions_` 和 `cmd_velocities_` 传递命令
   - 这是 ros2_control 的标准接口设计

2. **在 write() 函数中发送**
   - `write()` 函数每个控制周期被调用一次
   - 在这里读取控制器的命令并发送给下位机

## 已修改的代码

### 位置
**文件：** `src/tide_control/tide_hw_interface/src/tide_hw_interface.cpp`  
**函数：** `TideHardwareInterface::write()`  
**行号：** 约 1009-1070 行

### 当前实现（方式1：只发送浮点数）

```cpp
hardware_interface::return_type TideHardwareInterface::write(...)
{
  if (need_calibration_ || enable_virtual_control_)
  {
    return hardware_interface::return_type::OK;
  }

  // ========== 发送自定义数据给下位机 ==========
  
  std::vector<float> float_data;
  
  // 添加所有关节的位置和速度
  for (size_t i = 0; i < joint_count; i++)
  {
    double position_cmd = 0.0;
    double velocity_cmd = 0.0;
    
    // 从控制器获取命令
    if (!std::isnan(cmd_positions_[i]) && 
        info_.joints[i].command_interfaces[0].name == "position")
    {
      position_cmd = cmd_positions_[i];
    }
    if (!std::isnan(cmd_velocities_[i]) && 
        info_.joints[i].command_interfaces[0].name == "velocity")
    {
      velocity_cmd = cmd_velocities_[i];
    }
    
    // 添加到数据数组
    float_data.push_back(static_cast<float>(position_cmd));
    float_data.push_back(static_cast<float>(velocity_cmd));
  }
  
  // 发送浮点数数组（自动打包）
  sendFloatArray(float_data);
  
  return hardware_interface::return_type::OK;
}
```

### 发送的数据格式

假设有 4 个关节：

**你的数据（8个浮点数）：**
```
position[0], velocity[0],  // 关节1
position[1], velocity[1],  // 关节2
position[2], velocity[2],  // 关节3
position[3], velocity[3]   // 关节4
总计：8个float = 32字节
```

**自动打包后的完整包：**
```
0xFF 0xAA          # 帧头 (2字节)
20 00              # 长度=32 (2字节)
[32字节浮点数据]   # 你的数据
XX XX XX XX        # CRC32 (4字节)
总计：40字节
```

## 如何切换到混合数据（浮点数 + 整数）

如果需要发送混合数据，取消注释方式2的代码：

```cpp
hardware_interface::return_type TideHardwareInterface::write(...)
{
  // 注释掉方式1
  // std::vector<float> float_data;
  // ...
  // sendFloatArray(float_data);
  
  // 使用方式2：混合数据
  PacketBuilder builder;
  
  // 添加数据包类型
  builder.addByte(0x01);
  
  // 添加时间戳
  builder.addUInt32(static_cast<uint32_t>(
    std::chrono::system_clock::now().time_since_epoch().count() / 1000000
  ));
  
  // 添加关节数据
  for (size_t i = 0; i < joint_count; i++)
  {
    double position_cmd = 0.0;
    double velocity_cmd = 0.0;
    
    if (!std::isnan(cmd_positions_[i]) && 
        info_.joints[i].command_interfaces[0].name == "position")
    {
      position_cmd = cmd_positions_[i];
    }
    if (!std::isnan(cmd_velocities_[i]) && 
        info_.joints[i].command_interfaces[0].name == "velocity")
    {
      velocity_cmd = cmd_velocities_[i];
    }
    
    builder.addFloat(static_cast<float>(position_cmd))
           .addFloat(static_cast<float>(velocity_cmd));
  }
  
  // 添加状态标志
  builder.addUInt16(0x0001);
  
  // 发送
  sendCustomData(builder.getData());
  
  return hardware_interface::return_type::OK;
}
```

### 混合数据格式

**你的数据：**
```
类型(1字节) + 时间戳(4字节) + 8个浮点数(32字节) + 状态(2字节)
= 39字节
```

**自动打包后：**
```
0xFF 0xAA          # 帧头 (2字节)
27 00              # 长度=39 (2字节)
[39字节混合数据]   # 你的数据
XX XX XX XX        # CRC32 (4字节)
总计：47字节
```

```cpp
class MyController : public controller_interface::ControllerInterface
{
public:
  controller_interface::return_type update(
      const rclcpp::Time& time,
      const rclcpp::Duration& period) override
  {
    // 控制器只需要设置命令
    joint_position_command_[0] = 1.23;  // 设置关节1位置
    joint_velocity_command_[0] = 0.5;   // 设置关节1速度
    
    // 这些命令会自动传递到 TideHardwareInterface::write()
    // 然后在 write() 中发送给下位机

  }
};
```

## 发送频率

- `write()` 函数的调用频率由 ros2_control 的更新频率决定
- 通常是 100Hz - 1000Hz
- 在 URDF 中配置：

```xml
<ros2_control name="TideHardwareInterface" type="system">
  <hardware>
    <plugin>tide_hw_interface/TideHardwareInterface</plugin>
    <param name="update_rate">100</param>  <!-- 100Hz -->
  </hardware>
</ros2_control>
```

## 常见问题

### Q1: 控制器的命令没有发送到下位机？

**检查：**
1. 确认 `write()` 函数被调用（添加日志）
2. 确认 `enable_virtual_control_` 为 false
3. 确认 `need_calibration_` 为 false
4. 确认串口已正确初始化

### Q2: 如何发送额外的数据（如IMU）？

```cpp
hardware_interface::return_type TideHardwareInterface::write(...)
{
  PacketBuilder builder;
  
  // 关节数据
  for (size_t i = 0; i < joint_count; i++)
  {
    builder.addFloat(static_cast<float>(state_positions_[i]))
           .addFloat(static_cast<float>(state_velocities_[i]));
  }
  
  // IMU数据
  builder.addFloat(static_cast<float>(imu_rpy_[0]))
         .addFloat(static_cast<float>(imu_rpy_[1]))
         .addFloat(static_cast<float>(imu_rpy_[2]));
  
  sendCustomData(builder.getData());
  
  return hardware_interface::return_type::OK;
}
```

### Q3: 如何降低发送频率？

```cpp
hardware_interface::return_type TideHardwareInterface::write(...)
{
  static int counter = 0;
  counter++;
  
  // 每10次调用发送一次（降低到原来的1/10）
  if (counter % 10 == 0)
  {
    std::vector<float> float_data;
    // ... 构建数据
    sendFloatArray(float_data);
  }
  
  return hardware_interface::return_type::OK;
}
```

✅ **控制器 → write() → 发送函数 → 下位机**

✅ **自动打包（帧头 + 长度 + CRC32）**  
✅ **每个控制周期自动发送**
