# 电机数据存储说明

## 功能说明

`parseMotorData`函数已经实现了完整的数据帧解析功能，将接收到的串口数据处理后存储到`motor_data_buffer_`数组中，供你后续自定义解析。

## 数据流程

```
串口接收 → motor_rx_buffer_ (原始数据)
         ↓
    查找帧头 0xFF 0xAA
         ↓
    读取数据长度字段
         ↓
    验证CRC32校验
         ↓
    提取纯净数据 → motor_data_buffer_ (去除帧头、长度、CRC32)
         ↓
    等待你自定义解析
```

## motor_data_buffer_ 说明

### 数据内容
- `motor_data_buffer_`是一个`std::vector<uint8_t>`类型的成员变量
- 存储的是纯净的数据，已去除：
  - 帧头（0xFF 0xAA）
  - 长度字段（2字节）
  - CRC32校验码（4字节）

### 数据特点
- 每次接收到新的有效数据包时，会先`clear()`清空旧数据
- 然后插入新的纯净数据
- 数据大小由下位机发送的`data_length`字段决定

## 使用方法

### 在read()函数中解析

你可以在`read()`函数中访问`motor_data_buffer_`并按照自己的协议解析：

```cpp
hardware_interface::return_type TideHardwareInterface::read(
    const rclcpp::Time& time, const rclcpp::Duration& period)
{
  // 检查是否有新数据
  if (!motor_data_buffer_.empty())
  {
    // 方式1：按字节解析
    size_t offset = 0;
    while (offset < motor_data_buffer_.size())
    {
      uint8_t motor_id = motor_data_buffer_[offset++];
      
      float position;
      std::memcpy(&position, motor_data_buffer_.data() + offset, sizeof(float));
      offset += sizeof(float);
      
      // ... 继续解析其他数据
    }
    
    // 方式2：按结构体解析
    struct MotorData {
      uint8_t id;
      float position;
      float velocity;
      float current;
      float temperature;
    } __attribute__((packed));
    
    size_t num_motors = motor_data_buffer_.size() / sizeof(MotorData);
    for (size_t i = 0; i < num_motors; ++i)
    {
      MotorData* data = reinterpret_cast<MotorData*>(
          motor_data_buffer_.data() + i * sizeof(MotorData));
      
      // 使用data->id, data->position等
    }
  }
  
  return hardware_interface::return_type::OK;
}
```

### 在其他函数中解析

你也可以创建一个专门的解析函数：

```cpp
void TideHardwareInterface::parseMotorFeedback()
{
  if (motor_data_buffer_.empty()) return;
  
  // 你的解析逻辑
  // ...
}
```

然后在`read()`函数中调用：

```cpp
hardware_interface::return_type TideHardwareInterface::read(
    const rclcpp::Time& time, const rclcpp::Duration& period)
{
  parseMotorFeedback();
  return hardware_interface::return_type::OK;
}
```

## 数据包示例

### 下位机发送的完整数据包

```
+--------+--------+--------+--------+------------------+--------+
| 0xFF   | 0xAA   | Length | Length | Motor Data       | CRC32  |
|        |        | (Low)  | (High) | (N bytes)        | (4B)   |
+--------+--------+--------+--------+------------------+--------+
```

### motor_data_buffer_ 中的数据

```
+------------------+
| Motor Data       |
| (N bytes)        |
+------------------+
```

只包含中间的数据部分，帧头、长度、CRC32都已被去除。

## 注意事项

1. **线程安全**：`motor_data_buffer_`在串口接收线程中被更新，在`read()`函数（主线程）中被读取。如果需要，可以添加互斥锁保护。

2. **数据覆盖**：每次接收到新数据包时，旧数据会被清空。如果你需要保留历史数据，请在解析后自行保存。

3. **数据大小**：`motor_data_buffer_.size()`返回数据字节数，可用于判断数据是否完整。

4. **字节序**：确保上位机和下位机使用相同的字节序（通常都是小端序）。

## 协议自定义示例

### 示例1：简单的ID+值格式

```
数据格式：[ID(1字节)][Value(4字节float)] × N个电机
```

解析代码：
```cpp
size_t offset = 0;
while (offset + 5 <= motor_data_buffer_.size())
{
  uint8_t id = motor_data_buffer_[offset++];
  float value;
  std::memcpy(&value, motor_data_buffer_.data() + offset, 4);
  offset += 4;
  
  // 使用id和value
}
```

### 示例2：复杂的多字段格式

```
数据格式：[ID(1)][Pos(4)][Vel(4)][Cur(4)][Temp(4)] × N个电机
```

解析代码：
```cpp
size_t offset = 0;
const size_t motor_size = 17; // 1 + 4*4
while (offset + motor_size <= motor_data_buffer_.size())
{
  uint8_t id = motor_data_buffer_[offset++];
  
  float pos, vel, cur, temp;
  std::memcpy(&pos, motor_data_buffer_.data() + offset, 4); offset += 4;
  std::memcpy(&vel, motor_data_buffer_.data() + offset, 4); offset += 4;
  std::memcpy(&cur, motor_data_buffer_.data() + offset, 4); offset += 4;
  std::memcpy(&temp, motor_data_buffer_.data() + offset, 4); offset += 4;
  
  // 使用这些数据
}
```

### 示例3：混合数据格式

```
数据格式：[电机数量(1)][电机数据...][IMU数据...][其他数据...]
```

解析代码：
```cpp
size_t offset = 0;
uint8_t motor_count = motor_data_buffer_[offset++];

// 解析电机数据
for (uint8_t i = 0; i < motor_count; ++i)
{
  // 解析单个电机
}

// 解析IMU数据
// ...

// 解析其他数据
// ...
```

## 相关变量

- `motor_rx_buffer_`：原始接收缓冲区（包含帧头、长度、CRC32）
- `motor_data_buffer_`：纯净数据缓冲区（只包含有效数据）
- `motors_`：电机对象数组
- `state_positions_`、`state_velocities_`等：状态接口变量
