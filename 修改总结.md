# 电机串口通信协议修改总结

## 修改内容

### 1. 协议变更
- ✅ 帧头：从旧协议改为 `0xFF 0xAA`
- ✅ 校验：使用 CRC32 算法（替代 CRC16）
- ✅ 数据格式：支持灵活的数据格式，可传输几百个字节
- ✅ IMU协议：保持不变（`0x55 0xAA` + CRC16）
- ✅ 串口配置：电机和IMU必须使用独立的串口，不支持共享

### 2. 代码修改

#### 头文件 (tide_hw_interface.hpp)
- 添加了 `motor_data_buffer_` 成员变量，用于存储解析后的纯净数据
- 简化了数据结构，只保留必要的控制和反馈包头定义
- 添加了 `calculateCRC32()` 函数声明

#### 实现文件 (tide_hw_interface.cpp)
- 添加了 CRC32 查找表和计算函数
- 实现了 `parseMotorData()` 函数：
  - 自动查找帧头 `0xFF 0xAA`
  - 自动读取数据长度
  - 自动验证 CRC32
  - 将有效数据提取到 `motor_data_buffer_`
- 修改了串口初始化，强制使用独立串口（电机和IMU不能共享）
- 修改了 `write()` 函数，使用新协议发送控制指令
- 修改了 `stopMotors()` 函数，使用新协议

### 3. 文档和示例
- `MOTOR_PROTOCOL.md`：详细的协议说明文档

## 串口配置

### 重要说明
**电机和IMU必须使用独立的串口，不支持共享！**

### 配置示例
在你的 URDF 或 ros2_control 配置文件中：

```xml
<hardware>
  <plugin>tide_hw_interface/TideHardwareInterface</plugin>
  <param name="serial_port">/dev/ttyUSB0</param>          <!-- 电机串口 -->
  <param name="baud_rate">921600</param>
  <param name="imu_serial_port">/dev/ttyUSB1</param>      <!-- IMU串口（必须不同） -->
  <param name="imu_baud_rate">115200</param>
  <param name="enable_virtual_control">false</param>
  <param name="need_calibration">false</param>
</hardware>
```

### 串口说明
- `serial_port`: 电机通信串口
  - 发送：控制指令（0xFF 0xAA + CRC32）
  - 接收：电机反馈数据（0xFF 0xAA + CRC32）
  
- `imu_serial_port`: IMU通信串口
  - 只接收：IMU数据（0x55 0xAA + CRC16）
  - 必须与电机串口不同

## 协议格式

### 控制包（上位机 → 下位机）
```
帧头(2) | 长度(2) | 电机ID(1) | 位置(4) | 速度(4) | 力矩(4) | CRC32(4)
0xFF 0xAA | data_length | motor_id | position | velocity | torque | crc32
总长度：21字节
```

### 反馈包（下位机 → 上位机）
```
帧头(2) | 长度(2) | 数据(N) | CRC32(4)
0xFF 0xAA | data_length | [自定义数据] | crc32
总长度：8 + data_length 字节
```

**重要**：反馈包的数据格式完全由你自定义！

## 使用方法

### 1. 发送控制指令（已实现）
代码已经在 `write()` 函数中实现了发送逻辑，会自动：
- 构建控制包
- 计算 CRC32
- 通过串口发送

### 2. 接收反馈数据（需要你实现）

#### 步骤1：下位机发送数据
下位机按照以下格式发送数据：
```cpp
// 示例：发送3个电机的数据
uint8_t buffer[100];
size_t offset = 0;

// 帧头
buffer[offset++] = 0xFF;
buffer[offset++] = 0xAA;

// 数据长度（例如：3个电机 × 17字节 = 51字节）
uint16_t data_length = 51;
memcpy(buffer + offset, &data_length, 2);
offset += 2;

// 电机1数据
buffer[offset++] = 1; // motor_id
float pos1 = 1.57f, vel1 = 0.5f, cur1 = 2.3f, temp1 = 45.5f;
memcpy(buffer + offset, &pos1, 4); offset += 4;
memcpy(buffer + offset, &vel1, 4); offset += 4;
memcpy(buffer + offset, &cur1, 4); offset += 4;
memcpy(buffer + offset, &temp1, 4); offset += 4;

// 电机2、3数据...（类似）

// CRC32
uint32_t crc = calculateCRC32(buffer, offset);
memcpy(buffer + offset, &crc, 4);
offset += 4;

// 发送
serial_write(buffer, offset);
```

#### 步骤2：上位机解析数据
在 `tide_hw_interface.cpp` 的 `parseMotorData()` 函数中，找到这行注释：
```cpp
// TODO: 在这里添加你自己的数据解析逻辑
```

在这里添加你的解析代码，例如：
```cpp
// motor_data_buffer_ 已经包含了纯净数据（不含帧头、长度、CRC32）
size_t offset = 0;
while (offset + 17 <= motor_data_buffer_.size())
{
  uint8_t motor_id = motor_data_buffer_[offset++];
  
  float position, velocity, current, temperature;
  memcpy(&position, motor_data_buffer_.data() + offset, 4); offset += 4;
  memcpy(&velocity, motor_data_buffer_.data() + offset, 4); offset += 4;
  memcpy(&current, motor_data_buffer_.data() + offset, 4); offset += 4;
  memcpy(&temperature, motor_data_buffer_.data() + offset, 4); offset += 4;
  
  // 更新电机状态
  for (auto& motor : motors_)
  {
    if (motor->config_.tx_id == motor_id)
    {
      motor->angle_current = position;
      motor->measure.speed_aps = velocity;
      motor->measure.real_current = current;
      motor->measure.temperature = temperature;
      motor->status = MOTOR_OK;
      motor->update_timestamp(rclcpp::Clock().now());
      break;
    }
  }
}
```

## 常见问题

### Q1: 如何确定数据长度？
A: `data_length` 字段表示纯数据的长度，不包括帧头(2)、长度字段(2)和CRC32(4)。
例如：发送51字节数据，`data_length = 51`，总包长度 = 2 + 2 + 51 + 4 = 59字节。

### Q2: 大小端问题？
A: 默认使用小端序（Little Endian）。如果下位机使用大端序，需要进行字节序转换。

### Q3: 如何处理数据丢失？
A: 代码会自动丢弃CRC校验失败的包。建议在应用层实现超时重传机制。

### Q4: 可以混合使用整数和浮点数吗？
A: 可以！数据格式完全自定义，只要上下位机约定一致即可。

### Q5: IMU数据会受影响吗？
A: 不会！IMU使用独立的协议（`0x55 0xAA` + CRC16），完全不受影响。
