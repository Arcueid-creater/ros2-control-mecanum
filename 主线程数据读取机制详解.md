# 主线程如何读取串口数据详解

## 核心机制：共享内存 + 回调函数

主线程**不直接读取串口**，而是通过**共享内存变量**读取数据。串口线程负责更新这些共享变量。

---

## 完整数据流程

### 1. 初始化阶段：建立连接

```cpp
// 在 TideHardwareInterface::on_activate() 中

// ============ 电机串口初始化 ============
serial_device_ = std::make_unique<SerialDevice>(
    serial_port_,           // 串口设备路径
    baud_rate_,            // 波特率
    [this](const uint8_t* data, size_t len) {  // ← 回调函数
        this->parseMotorData(data, len);       // 解析电机数据
    }
);

// ============ IMU串口初始化 ============
imu_serial_device_ = std::make_unique<SerialDevice>(
    imu_serial_port_,      // IMU串口设备路径
    imu_baud_rate_,        // 波特率
    [this](const uint8_t* data, size_t len) {  // ← 回调函数
        this->parseImuData(data, len);         // 解析IMU数据
    }
);
```

**关键点**：
- 创建了2个独立的 `SerialDevice` 对象
- 每个对象都有自己的回调函数
- 回调函数是成员函数，可以访问类的成员变量（共享内存）

---

### 2. 运行阶段：数据流转

#### 步骤1：串口线程读取数据

```cpp
// 在 SerialDevice::run() 中（独立线程）
void SerialDevice::run()
{
    while (!stop_)
    {
        struct pollfd fds[1];
        fds[0].fd = fd_;           // 串口文件描述符
        fds[0].events = POLLIN;    // 监听可读事件
        
        // 阻塞等待数据（最多10ms）
        int ret = poll(fds, 1, 10);
        
        if (ret > 0 && (fds[0].revents & POLLIN))
        {
            uint8_t buffer[1024];
            // 从串口读取数据
            ssize_t n = read(fd_, buffer, sizeof(buffer));
            
            if (n > 0)
            {
                // 调用回调函数！
                if (callback_)
                {
                    callback_(buffer, n);  // ← 这里调用回调
                }
            }
        }
    }
}
```

**关键点**：
- 这段代码在**独立线程**中运行
- `poll()` 阻塞等待串口数据，不影响主线程
- 收到数据后立即调用回调函数

---

#### 步骤2：回调函数更新共享内存

##### 电机数据回调

```cpp
// 在 TideHardwareInterface::parseMotorData() 中
// 这个函数在电机串口线程中执行！
void TideHardwareInterface::parseMotorData(const uint8_t* data, size_t len)
{
    // 解析数据...
    
    // 更新共享内存变量（电机状态）
    for (auto& motor : motors_)
    {
        if (motor.can_id == parsed_id)
        {
            // 写入共享内存！
            motor.position = parsed_position;      // ← 线程2写入
            motor.velocity = parsed_velocity;      // ← 线程2写入
            motor.effort = parsed_effort;          // ← 线程2写入
            break;
        }
    }
}
```

##### IMU数据回调

```cpp
// 在 TideHardwareInterface::parseImuData() 中
// 这个函数在IMU串口线程中执行！
void TideHardwareInterface::parseImuData(const uint8_t* frame)
{
    ImuPacket packet;
    std::memcpy(&packet, frame, sizeof(ImuPacket));
    
    // 根据数据类型更新不同的共享内存变量
    if (packet.rid == 0x03) // RPY数据
    {
        // 写入共享内存！
        imu_rpy_[0] = packet.data[0] * M_PI / 180.0;  // ← 线程3写入
        imu_rpy_[1] = packet.data[1] * M_PI / 180.0;  // ← 线程3写入
        imu_rpy_[2] = packet.data[2] * M_PI / 180.0;  // ← 线程3写入
    }
    else if (packet.rid == 0x02) // 角速度数据
    {
        imu_gyro_[0] = packet.data[0] * M_PI / 180.0;  // ← 线程3写入
        imu_gyro_[1] = packet.data[1] * M_PI / 180.0;
        imu_gyro_[2] = packet.data[2] * M_PI / 180.0;
    }
    else if (packet.rid == 0x01) // 加速度数据
    {
        imu_accel_[0] = packet.data[0];  // ← 线程3写入
        imu_accel_[1] = packet.data[1];
        imu_accel_[2] = packet.data[2];
    }
    else if (packet.rid == 0x04) // 四元数数据
    {
        imu_orientation[0] = packet.data[0];  // w  ← 线程3写入
        imu_orientation[1] = packet.data[1];  // x
        imu_orientation[2] = packet.data[2];  // y
        imu_orientation[3] = packet.data[3];  // z
    }
}
```

**关键点**：
- 回调函数在**串口线程**中执行（不是主线程！）
- 回调函数更新类的成员变量（共享内存）
- 这些变量可以被主线程访问

---

#### 步骤3：主线程读取共享内存

```cpp
// 在 TideHardwareInterface::read() 中
// 这个函数在主线程中执行！
hardware_interface::return_type TideHardwareInterface::read(
    const rclcpp::Time& time, const rclcpp::Duration& period)
{
    // ============ 读取电机数据（从共享内存） ============
    // motors_ 已经被电机串口线程更新了
    // 这里只是读取，不需要做任何串口操作
    
    // 电机数据已经在 parseMotorData() 中更新到 motors_ 向量中
    // ros2_control 会自动从 motors_[i].position 等变量读取
    
    // ============ 读取IMU数据（从共享内存） ============
    // imu_rpy_, imu_gyro_, imu_accel_, imu_orientation
    // 已经被IMU串口线程更新了
    
    // 发布RPY话题
    if (publish_rpy_ && rpy_pub_)
    {
        auto rpy_msg = geometry_msgs::msg::Vector3Stamped();
        rpy_msg.header.stamp = nh_->now();
        rpy_msg.header.frame_id = imu_frame_id_;
        
        // 从共享内存读取！
        rpy_msg.vector.x = imu_rpy_[0] * 180.0 / M_PI;  // ← 线程1读取
        rpy_msg.vector.y = imu_rpy_[1] * 180.0 / M_PI;  // ← 线程1读取
        rpy_msg.vector.z = imu_rpy_[2] * 180.0 / M_PI;  // ← 线程1读取
        
        rpy_pub_->publish(rpy_msg);
    }
    
    // IMU状态接口也是从共享内存读取
    // export_state_interfaces() 中已经绑定了指针：
    // interfaces.emplace_back("imu_sensor", "rpy.roll", &imu_rpy_[0]);
    // 所以 ros2_control 会自动读取 imu_rpy_[0] 的值
    
    return hardware_interface::return_type::OK;
}
```

**关键点**：
- 主线程**不调用任何串口读取函数**
- 主线程只是**读取共享内存变量**
- 这些变量已经被串口线程更新好了

## 共享内存变量详解

### 在头文件中定义

```cpp
// 在 tide_hw_interface.hpp 中
class TideHardwareInterface : public hardware_interface::SystemInterface
{
private:
    // ============ 电机共享内存 ============
    struct Motor {
        std::string name;
        int can_id;
        double position;    // ← 线程2写，线程1读
        double velocity;    // ← 线程2写，线程1读
        double effort;      // ← 线程2写，线程1读
        double cmd_position;
        double cmd_velocity;
        // ...
    };
    std::vector<Motor> motors_;  // ← 共享内存
    
    // ============ IMU共享内存 ============
    double imu_rpy_[3] = {0.0, 0.0, 0.0};        // ← 线程3写，线程1读
    double imu_gyro_[3] = {0.0, 0.0, 0.0};       // ← 线程3写，线程1读
    double imu_accel_[3] = {0.0, 0.0, 0.0};      // ← 线程3写，线程1读
    double imu_orientation[4] = {0.0, 0.0, 0.0, 0.0};  // ← 线程3写，线程1读
    
    // ============ 串口设备（独立线程） ============
    std::unique_ptr<SerialDevice> serial_device_;      // 电机串口线程
    std::unique_ptr<SerialDevice> imu_serial_device_;  // IMU串口线程
};
```

## 完整时序图

```
时间 →

线程2(电机串口)          线程3(IMU串口)           线程1(主线程)
     │                       │                       │
     │ poll()                │ poll()                │
     │ 阻塞等待...            │ 阻塞等待...            │
     │                       │                       │
     ├─ 收到电机数据          │                       │
     │                       │                       │
     ├─ callback()           │                       │
     │  parseMotorData()     │                       │
     │  ↓                    │                       │
     │  motors_[0].position  │                       │
     │  = 1.23 ← 写入共享内存 │                       │
     │                       │                       │
     │                       ├─ 收到IMU数据           │
     │                       │                       │
     │                       ├─ callback()           │
     │                       │  parseImuData()       │
     │                       │  ↓                    │
     │                       │  imu_rpy_[0] = 0.5    │
     │                       │  ← 写入共享内存        │
     │                       │                       │
     │                       │                       ├─ read() 被调用
     │                       │                       │  (周期性，如100Hz)
     │                       │                       │
     │                       │                       ├─ 读取共享内存
     │                       │                       │  pos = motors_[0].position
     │                       │                       │  (读到 1.23)
     │                       │                       │
     │                       │                       ├─ 读取共享内存
     │                       │                       │  roll = imu_rpy_[0]
     │                       │                       │  (读到 0.5)
     │                       │                       │
     │                       │                       ├─ 发布话题
     │                       │                       │  /imu/rpy
     │                       │                       │
     │ poll()                │ poll()                │
     │ 阻塞等待...            │ 阻塞等待...            │
     ↓                       ↓                       ↓
```

## 为什么这样设计？

### 优点

1. **非阻塞**：主线程不需要等待串口数据
   ```cpp
   // 如果主线程直接读串口（不好的设计）：
   read(serial_fd, buffer, size);  // ← 会阻塞主线程！
   ```

2. **实时性**：主线程可以保持固定频率运行
   ```cpp
   // 主线程每10ms执行一次 read()
   // 不管串口有没有新数据，都能按时执行
   ```

3. **并发处理**：两个串口可以同时读取
   ```cpp
   // 线程2和线程3可以同时工作
   // 不会互相阻塞
   ```

4. **简单高效**：主线程只需要读取内存变量
   ```cpp
   // 读取内存变量非常快（纳秒级）
   double roll = imu_rpy_[0];  // ← 极快！
   
   // 而读取串口很慢（毫秒级）
   read(fd, buffer, size);     // ← 很慢！
   ```

## 代码追踪示例

### 追踪IMU数据的完整路径

```cpp
// ========== 步骤1：硬件发送数据 ==========
// IMU硬件 → 串口 /dev/ttyUSB0

// ========== 步骤2：串口线程读取 ==========
// 在 SerialDevice::run() 中（线程3）
int ret = poll(fds, 1, 10);  // 等待数据
ssize_t n = read(fd_, buffer, sizeof(buffer));  // 读取串口
callback_(buffer, n);  // 调用回调

// ========== 步骤3：回调函数解析并写入共享内存 ==========
// 在 TideHardwareInterface::parseImuData() 中（线程3）
void TideHardwareInterface::parseImuData(const uint8_t* frame)
{
    ImuPacket packet;
    std::memcpy(&packet, frame, sizeof(ImuPacket));
    
    if (packet.rid == 0x03) // RPY
    {
        imu_rpy_[0] = packet.data[0] * M_PI / 180.0;  // ← 写入
        imu_rpy_[1] = packet.data[1] * M_PI / 180.0;
        imu_rpy_[2] = packet.data[2] * M_PI / 180.0;
    }
}

// ========== 步骤4：主线程读取共享内存 ==========
// 在 TideHardwareInterface::read() 中（线程1）
hardware_interface::return_type TideHardwareInterface::read(...)
{
    // 读取共享内存
    rpy_msg.vector.x = imu_rpy_[0] * 180.0 / M_PI;  // ← 读取
    rpy_msg.vector.y = imu_rpy_[1] * 180.0 / M_PI;
    rpy_msg.vector.z = imu_rpy_[2] * 180.0 / M_PI;
    
    // 发布话题
    rpy_pub_->publish(rpy_msg);
}

// ========== 步骤5：状态接口自动读取 ==========
// ros2_control 框架自动读取
// 因为在 export_state_interfaces() 中绑定了指针：
interfaces.emplace_back("imu_sensor", "rpy.roll", &imu_rpy_[0]);
//                                                  ↑
//                                            直接指向共享内存

// ========== 步骤6：broadcaster读取状态接口并发布 ==========
// IMUSensorBroadcaster::update() 中
double roll = state_interfaces_[0].get_value();  // ← 读取状态接口
// 实际上就是读取 imu_rpy_[0]
```

## 关键概念总结

### 1. 主线程不直接读串口
```cpp
// ❌ 主线程不这样做：
read(serial_fd, buffer, size);  // 会阻塞！

// ✅ 主线程这样做：
double value = shared_variable;  // 读取共享内存，极快！
```

### 2. 串口线程负责更新共享内存
```cpp
// 在串口线程中：
callback_(buffer, n);  // → parseImuData() → 更新 imu_rpy_[]
```

### 3. 共享内存是桥梁
```cpp
// 串口线程写：
imu_rpy_[0] = new_value;  // 线程3写入

// 主线程读：
double roll = imu_rpy_[0];  // 线程1读取
```

### 4. 回调函数是关键
```cpp
// 回调函数连接了串口线程和共享内存
SerialDevice(..., [this](data, len) {
    this->parseImuData(data, len);  // 在串口线程中执行
});
```

## 类比理解

想象一个快递系统：

- **串口线程** = 快递员（在外面跑，收集包裹）
- **共享内存** = 快递柜（存放包裹的地方）
- **主线程** = 收件人（定期去快递柜取包裹）

**流程**：
1. 快递员收到包裹（串口收到数据）
2. 快递员放入快递柜（回调函数更新共享内存）
3. 收件人定期去快递柜取包裹（主线程读取共享内存）

**好处**：
- 收件人不需要等快递员（主线程不阻塞）
- 快递员可以继续送其他包裹（串口线程继续工作）
- 收件人可以按自己的节奏取包裹（主线程按固定频率运行）
