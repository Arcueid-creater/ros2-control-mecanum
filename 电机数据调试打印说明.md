# 电机数据调试打印说明

## 功能说明

在`parseMotorData`函数中添加了调试打印功能，每次接收到有效的电机数据包后，会自动将`motor_data_buffer_`中的纯净数据以十六进制格式打印到终端。

## 打印格式

### 输出示例

```
[INFO] [tide_hw_interface]: Motor data [51 bytes]: 01 00 00 c8 42 00 00 00 3f 00 00 13 40 00 00 36 42 
                                      02 00 00 47 42 00 00 9a 3e 00 00 66 3f 00 00 28 42 
                                      03 00 00 16 43 00 00 cc 3f 00 00 46 40 00 00 41 42
```

### 格式说明

- 第一行显示数据总字节数：`[51 bytes]`
- 每个字节以两位十六进制显示：`01 00 00 c8 42`
- 每16字节自动换行，方便查看
- 使用`RCLCPP_INFO`输出，可以在终端和日志文件中查看

## 数据解读

### 示例数据解析

假设协议格式为：`[ID(1字节)][Position(4字节)][Velocity(4字节)][Current(4字节)][Temperature(4字节)]`

```
01 00 00 c8 42 00 00 00 3f 00 00 13 40 00 00 36 42
│  └─────┬────┘ └─────┬────┘ └─────┬────┘ └─────┬────┘
│     Position     Velocity     Current    Temperature
ID=1
```

转换为实际值：
- ID: 0x01 = 1
- Position: 0x4200c800 = 100.390625 (float)
- Velocity: 0x3f000000 = 0.5 (float)
- Current: 0x40130000 = 2.296875 (float)
- Temperature: 0x42360000 = 45.5 (float)

### 使用Python快速解析

```python
import struct

# 从终端复制的十六进制数据
hex_data = "01 00 00 c8 42 00 00 00 3f 00 00 13 40 00 00 36 42"
bytes_data = bytes.fromhex(hex_data.replace(" ", ""))

# 解析
motor_id = bytes_data[0]
position = struct.unpack('<f', bytes_data[1:5])[0]
velocity = struct.unpack('<f', bytes_data[5:9])[0]
current = struct.unpack('<f', bytes_data[9:13])[0]
temperature = struct.unpack('<f', bytes_data[13:17])[0]

print(f"Motor ID: {motor_id}")
print(f"Position: {position}")
print(f"Velocity: {velocity}")
print(f"Current: {current}")
print(f"Temperature: {temperature}")
```


## 调试技巧

### 1. 验证数据长度

检查打印的字节数是否符合预期：
- 1个电机：17字节（1+4+4+4+4）
- 3个电机：51字节（17×3）
- N个电机：17×N字节

### 2. 检查数据对齐

每16字节换行，方便查看数据是否对齐：
```
01 00 00 c8 42 00 00 00 3f 00 00 13 40 00 00 36  ← 16字节
42 02 00 00 47 42 ...                              ← 下一行
```

### 3. 识别异常数据

- 全0数据：`00 00 00 00 00 00 00 00`
- 全FF数据：`ff ff ff ff ff ff ff ff`
- 重复数据：连续多次相同的数据包

### 4. 对比发送和接收

在下位机端也打印发送的数据，对比上位机接收的数据是否一致。

## 性能考虑

### 打印频率

如果数据更新频率很高（如1kHz），打印会影响性能。可以添加限流：

```cpp
// 每秒最多打印10次
static auto last_print_time = std::chrono::steady_clock::now();
auto now = std::chrono::steady_clock::now();
auto elapsed = std::chrono::duration_cast<std::chrono::milliseconds>(now - last_print_time);

if (elapsed.count() >= 100) // 100ms间隔
{
  // 打印代码
  last_print_time = now;
}
```

### 条件编译

可以使用宏控制是否打印：

```cpp
#define DEBUG_MOTOR_DATA 1

#if DEBUG_MOTOR_DATA
  // 打印代码
#endif
```

发布版本时设置为0即可关闭打印。

## 关闭打印

如果不需要打印，有两种方式：

### 方式1：注释掉打印代码

在`tide_hw_interface.cpp`中找到打印代码并注释：

```cpp
// 打印纯净数据（十六进制格式）
/*
if (!motor_data_buffer_.empty())
{
  // ... 打印代码
}
*/
```

### 方式2：使用日志级别过滤

ROS2可以通过日志级别过滤：

```bash
ros2 launch tide_ctrl_bringup tide_ctrl_bringup.launch.py \
  --ros-args --log-level tide_hw_interface:=WARN
```

这样只会显示WARN及以上级别的日志，INFO级别的打印会被过滤。

## 常见问题

### Q1: 打印的数据是乱码

A: 检查：
1. 串口波特率是否正确
2. CRC32校验是否通过
3. 帧头是否正确（0xFF 0xAA）

### Q2: 没有任何打印输出

A: 检查：
1. 串口是否连接
2. 下位机是否发送数据
3. 日志级别是否设置为INFO或更低

### Q3: 打印的字节数不对

A: 检查：
1. 下位机发送的`data_length`字段是否正确
2. 数据包是否完整
3. 是否有数据丢失

### Q4: 数据看起来不对

A: 
1. 确认字节序（大端/小端）
2. 确认数据类型（float/int）
3. 使用Python脚本验证解析
