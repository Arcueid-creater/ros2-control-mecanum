# 下位机代码对比说明

## 概述
新的下位机代码 (`trans_task_new.c/h`) 完整保留了原代码的线程间通信功能，同时实现了新的通信协议（0xFF 0xAA + CRC32）。

---

## 一、保留的功能（与原代码相同）

### 1. MCN线程间通信 ✅
**完全保留，功能一致**

#### 发布的话题
```c
MCN_DECLARE(transmission_fdb_topic);
static struct trans_fdb_msg trans_fdb_data;
```
- 发布函数：`trans_pub_push()` - 向其他线程发布传输反馈数据

#### 订阅的话题（5个）
```c
MCN_DECLARE(ins_topic);              // INS姿态数据
MCN_DECLARE(chassis_cmd);            // 底盘命令
MCN_DECLARE(gimbal_cmd);             // 云台命令
MCN_DECLARE(gimbal_fdb_topic);       // 云台反馈
MCN_DECLARE(gimbal_ins_topic);       // 云台IMU数据
```

#### 三个核心函数
1. **`trans_sub_init()`** - 初始化所有订阅
   - 订阅5个话题，获取节点句柄
   
2. **`trans_sub_pull()`** - 拉取订阅数据
   - 使用 `mcn_poll()` 检查数据更新
   - 使用 `mcn_copy()` 复制最新数据到本地变量
   
3. **`trans_pub_push()`** - 发布数据
   - 使用 `mcn_publish()` 发布 `trans_fdb_data`

### 2. 任务初始化和控制流程 ✅
```c
void trans_task_init(void)
{
    // 初始化接收状态机
    // 调用 trans_sub_init() 初始化订阅
}

void trans_control_task(void)
{
    // 1. 拉取订阅数据
    trans_sub_pull();
    
    // 2. 心跳检测
    // 3. 发送数据
    // 4. 发布数据
    trans_pub_push();
    
    // 5. 性能监测
}
```

### 3. 使用订阅数据 ✅
新代码中使用了订阅的数据进行发送：
```c
// 使用云台反馈和云台IMU数据
yaw_obs = gimbal_fdb.yaw_offset_angle - gim_ins.yaw;

// 使用INS数据
float data[] = {
    ins.pitch,           // 来自 ins_topic
    yaw_obs,             // 来自 gimbal_fdb_topic 和 gimbal_ins_topic
    0.0f,
    (float)team_color
};
send_float_array(data, 4);
```

## 二、改变的功能（新协议）

### 1. 通信协议 🔄
| 项目 | 原协议 | 新协议 |
|------|--------|--------|
| 帧头 | 0xFF (单字节) | 0xFF 0xAA (双字节) |
| 长度字段 | 固定LEN字段 | 2字节动态长度 |
| 校验算法 | SC+AC (和校验+附加校验) | CRC32 |
| 数据长度 | 固定（根据功能码） | 灵活（最多256字节） |

### 2. 数据包格式 🔄
**原格式：**
```
[HEAD][D_ADDR][ID][LEN][DATA...][SC][AC]
```

**新格式：**
```
[0xFF][0xAA][长度(2字节)][数据(N字节)][CRC32(4字节)]
```

### 3. 发送方式 🔄
**原代码：**
- 使用 `pack_Rpy()` 填充固定格式的结构体
- 使用 `Check_Rpy()` 计算和校验
- 发送固定大小的 `RpyTypeDef` 结构体

**新代码：**
- 提供3种便捷发送方式：
  1. `send_single_motor()` - 发送单个电机数据
  2. `send_multiple_motors()` - 批量发送多个电机
  3. `send_float_array()` - 发送浮点数数组（最简单）
- 自动计算CRC32
- 支持灵活数据长度

### 4. 接收解析 🔄
**原代码：**
- 基于固定帧格式的状态机
- 解析特定的功能码（GIMBAL, HEARTBEAT等）
- 固定的数据结构

**新代码：**
- 基于动态长度的状态机
- 自动验证CRC32
- 提取干净数据（去除帧头、长度、CRC）
- 用户可自定义解析逻辑

## 五、使用建议

### 1. 直接替换
新代码可以直接替换原代码，因为：
- 所有MCN通信接口保持一致
- 任务初始化和控制流程相同
- 对外接口函数名称相同（`trans_task_init`, `trans_control_task`, `process_usb_data`, `get_trans_fdb`）

### 2. 发送数据示例
```c
// 在 trans_control_task() 中已有示例：

// 方式1：发送浮点数数组（推荐，最简单）
float data[] = {ins.pitch, yaw_obs, 0.0f, (float)team_color};
send_float_array(data, 4);

// 方式2：发送单个电机
send_single_motor(1, position, velocity, current, temp);

// 方式3：批量发送多个电机
MotorData_t motors[3];
// ... 填充数据
send_multiple_motors(motors, 3);
```
